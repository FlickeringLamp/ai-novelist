# GitPython 路径前缀问题

## 问题描述

在存档点功能的 Git 状态获取中，发现同一个文件出现两次：
- `./.userignore`（带 `./` 前缀）
- `.userignore`（无前缀）

## 原因分析

GitPython 库在比较工作区与索引（`repo.index.diff(None)`）时，会以不同方式解析路径：
- 某些路径以 `./` 开头
- 相同文件也可能以无前缀形式返回

这是 GitPython 的特性，无法通过 Git 配置完全避免。

## 日志验证

```
[1] 2026-04-07 11:20:05,535 - backend.git.checkpoint_service - INFO - Diff item: a_path='./.userignore', b_path='./.userignore', change_type='M'
[1] 2026-04-07 11:20:05,536 - backend.git.checkpoint_service - INFO - Diff item: a_path='.userignore', b_path='.userignore', change_type='M'
```

同一次 diff 返回了同一个文件的两条记录。

## 解决方案

在代码层面对路径进行标准化和去重：

```python
changes = []
seen_paths = set()
for item in diff_items:
    # 选择正确的路径
    if item.change_type == 'D':
        file_path = item.a_path
    else:
        file_path = item.b_path if item.b_path else item.a_path
    
    # 标准化路径：移除 './' 前缀
    if file_path.startswith('./'):
        file_path = file_path[2:]
    
    # 去重
    if file_path in seen_paths:
        continue
    seen_paths.add(file_path)
    
    changes.append({"path": file_path, ...})
```

## 修复范围

涉及两个方法：
1. `get_status()` - 获取工作区状态（`repo.index.diff(None)`）
2. `get_checkpoint_diff()` - 获取存档点差异（`parent_commit.diff(commit)`）

## 附：关于 .gitignore 不生效的说明

如果看到类似下面的日志，说明文件已被 Git 追踪：

```
Diff item: a_path='./db/checkpoints.db', b_path='./db/checkpoints.db', change_type='M'
```

**原因**：文件虽然被 `.gitignore` 忽略，但曾经执行过 `git add`，已在版本控制中。

**解决**：需要手动从索引中移除：
```bash
git rm --cached db/checkpoints.db chromadb/chroma.sqlite3
```

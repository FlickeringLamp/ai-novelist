## 命名转换的原因

整个 IPC 通信流程分为三层：

### 1. **主进程**（[`electron.cjs:314`](frontend/electron.cjs:314)）
```javascript
ipcMain.handle('mcp-install', async (event, serverId, packageName) => {
  return await installNpmMCPServer(serverId, packageName);
});
```
- IPC 通道名称使用 **kebab-case**（短横线命名）：`'mcp-install'`
- 因为这是字符串，可以包含短横线

### 2. **预加载脚本**（[`preload.cjs:12`](frontend/preload.cjs:12)）
```javascript
contextBridge.exposeInMainWorld('electronAPI', {
  mcpInstall: (serverId, packageName) => ipcRenderer.invoke('mcp-install', serverId, packageName),
});
```
- 暴露给前端的 API 使用 **camelCase**（驼峰命名）：`mcpInstall`
- 内部调用 `ipcRenderer.invoke('mcp-install', ...)` 连接到主进程的 IPC 处理器

### 3. **渲染进程**（[`ServerListPanel.tsx:103`](frontend/src/components/mcp/ServerListPanel.tsx:103)）
```javascript
await window.electronAPI?.mcpInstall(serverId, packageName);
```
- 前端通过 `window.electronAPI.mcpInstall` 调用

## 为什么要转换？

**JavaScript 对象属性名不能包含短横线**，因为短横线会被解释为减号操作符：

```javascript
// ❌ 错误：语法错误
window.electronAPI.mcp-install

// ✅ 正确：使用驼峰命名
window.electronAPI.mcpInstall
```

这是 Electron 中的常见命名约定：
- **IPC 通道**：kebab-case（如 `'mcp-install'`、`'window-minimize'`）
- **JavaScript 属性**：camelCase（如 `mcpInstall`、`windowMinimize`）

# 问题描述：

在前端保存一个两行消息
再次打开就出现多余的空行了
而且只出现在刚新建的文件

后续在这两行文本后面添加内容，并不会再凭空添加空行
如果第一次输入的是三行，那么三行之间都会凭空添加空行


# 定位问题：
Windows 系统上，前端发送的 \r\n（Windows 换行符）在通过 aiofiles.open() 以文本模式写入文件时，被错误地转换成了 \n\n（两个换行符），导致保存后再次打开时出现多余空行。

解决方案：
在 backend/file/file_service.py 的 update_file() 函数中添加了换行符标准化处理：

# 将 \r\n 转换为 \n，避免Windows换行符问题
content = content.replace('\r\n', '\n')

这样无论前端发送 \r\n 还是 \n，都会统一保存为 \n，避免换行符重复问题。

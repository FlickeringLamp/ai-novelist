自定义 Hook = 把组件逻辑提取成可复用的函数，名字以 use 开头
可以在函数组件中使用状态(state)和副作用等 React 特性。

传统类组件用 this.state、componentDidMount 等
函数组件用 Hook：useState、useEffect、useCallback 等

等号左边：解构赋值 + 重命名
Hook 返回一个对象，包含多个属性和方法。用 {} 解构取出

为什么要用 Hook？
不复用（写死在组件里）的问题：

// MessageInputPanel.tsx 会变得很臃肿
const [isOpen, setIsOpen] = useState(false);
const [allPaths, setAllPaths] = useState([]);
const [filteredPaths, setFilteredPaths] = useState([]);
// ... 还有一堆 useEffect、useCallback

用 Hook 的好处：

代码复用 - 其他组件也能用 useFilePathAutocomplete
关注点分离 - 补全逻辑独立管理，主组件只负责渲染
易于测试 - 逻辑单独抽离，可以单独测试
可读性 - 主组件代码更简洁

————————————
Hook 是 React 的逻辑复用机制，不是"为了用而用"。

不用 Hook：逻辑和 UI 耦合，组件臃肿，无法复用
用 Hook：像搭积木一样组合逻辑，组件保持简洁
你项目中的 useFilePathAutocomplete 就是一个很好的实践，把"输入 @ 触发补全"这个独立功能完整地封装了起来。

Hook vs Utils 普通函数的区别

1. 能否使用 React API
Utils 普通函数	Hook

使用 useEffect	❌ 不能	✅ 能
使用 useRef	❌ 不能	✅ 能
使用 useDispatch	❌ 不能	✅ 能（需 react-redux）
组件生命周期	❌ 无关	✅ 可以监听

你的理解完全正确！
Hook 的本质就是：

一个没有 JSX 返回值的"虚拟组件"，只负责逻辑，不负责渲染。

对比
特性	组件（Component）	Hook（Custom Hook）
可以使用 useState	✅	✅
可以使用 useEffect	✅	✅
可以使用其他 Hook	✅	✅
有生命周期（mount/update/unmount）	✅	✅
返回 JSX	✅	❌（返回数据/方法）
在 DOM 中创建节点	✅	❌
可以被 React DevTools 看到	✅	✅
形象比喻
组件 = 完整的工人（手 + 脑子）
       ↓
       既思考（逻辑）又干活（渲染 UI）

Hook = 只有脑子的顾问
       ↓
       只思考（逻辑），不干活（不碰 UI）
       多个工人可以共用同一个顾问

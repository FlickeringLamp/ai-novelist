electron的主文件，preload预加载脚本，都推荐cjs，
但是主文件也能使用mjs,

出现新的报错：

VM4 sandbox_bundle:2 Unable to load preload script: /home/denghuo/文档/ai-novelist/frontend/electron/preload.js
VM4 sandbox_bundle:2 SyntaxError: Cannot use import statement outside a module
    at runPreloadScript (VM4 sandbox_bundle:2:145240)
    at executeSandboxedPreloadScripts (VM4 sandbox_bundle:2:144535)
    at VM4 sandbox_bundle:2:157670
    at VM4 sandbox_bundle:2:157872
    at ___electron_webpack_init__ (VM4 sandbox_bundle:2:157876)
    at VM4 sandbox_bundle:2:157999

上网查找后，得知preload.js如果要用es模块
首先需要将后缀改为mjs,然后将所有引用的后缀也改成mjs,最后关闭沙盒模式
https://www.electronjs.org/zh/docs/latest/tutorial/esm#esm-preload-scripts-must-have-the-mjs-extension


**问题原因**：Electron 渲染进程默认启用沙盒模式，而沙盒化的预加载脚本不支持 ESM 导入（`import` 语句）。

**解决方案**：在 `windowManager.js` 第 21 行添加 `sandbox: false`，关闭沙盒模式。

**修改位置**：`frontend/electron/managers/windowManager.js`
```javascript
webPreferences: {
  nodeIntegration: false,
  contextIsolation: true,
  sandbox: false,  // 添加此行
  preload: APP_CONFIG.paths.preload,
},
```

现在预加载脚本 `preload.mjs` 的 ESM 导入可以正常加载了。

## Electron 前端架构解析

```
frontend/electron/

```

### 🔌 IPC 处理器是什么

**IPC (Inter-Process Communication)** 是 Electron 中主进程和渲染进程通信的机制。

| 组件 | 作用 | 代码位置 |
|------|------|----------|
| [`ipcMain`](frontend/electron/handlers/ipcHandlers.js:1) | 主进程端，监听并处理请求 | 主进程 |
| [`ipcRenderer`](frontend/electron/preload.mjs:1) | 渲染进程端，发送请求 | 预加载脚本 |
| [`contextBridge`](frontend/electron/preload.mjs:4) | 安全暴露 API 给前端 | 预加载脚本 |

**通信流程：**
```
前端 React 组件
    ↓ 调用 window.electronAPI.xxx()
preload.mjs (暴露安全 API)
    ↓ ipcRenderer.invoke()
ipcHandlers.js (ipcMain.handle() 注册处理器)
    ↓ 调用管理器方法
具体 Manager 执行操作
```

---

### 🏗️ 三个管理器 (Managers)

#### 1. [`WindowManager`](frontend/electron/managers/windowManager.js:4) - 窗口管理器
- **职责**：创建和控制应用窗口
- **功能**：
  - [`createWindow()`](frontend/electron/managers/windowManager.js:10) - 创建无边框主窗口
  - [`minimize()`](frontend/electron/managers/windowManager.js:54) / [`maximize()`](frontend/electron/managers/windowManager.js:61) / [`close()`](frontend/electron/managers/windowManager.js:72) - 窗口控制

#### 2. [`BackendManager`](frontend/electron/managers/backendManager.js:7) - 后端服务管理器
- **职责**：启动/停止 Python 后端服务
- **功能**：
  - [`start()`](frontend/electron/managers/backendManager.js:14) - 启动 Python 进程（开发模式）或 exe（打包模式）
  - [`stop()`](frontend/electron/managers/backendManager.js:50) - 停止后端服务
  - [`checkHealth()`](frontend/electron/managers/backendManager.js:101) - 健康检查
  - [`restart()`](frontend/electron/managers/backendManager.js:130) - 重启后端

#### 3. [`TerminalManager`](frontend/electron/managers/terminalManager.js:5) - 终端管理器
- **职责**：管理嵌入式终端（使用 node-pty）
- **功能**：
  - 创建 PTY 伪终端
  - 处理终端输入输出
  - 支持多终端实例管理

### 🤔 为什么终端 IPC 处理器放在管理器中注册

这是**架构设计的选择**，对比两种注册方式：

#### 方式一：简单功能 → 集中注册在 [`ipcHandlers.js`](frontend/electron/handlers/ipcHandlers.js:7)
```javascript
// 如窗口控制、后端重启等简单调用
ipcMain.handle('window-minimize', () => windowManager.minimize());
```

#### 方式二：复杂功能 → 在 Manager 内部注册 ([`TerminalManager`](frontend/electron/managers/terminalManager.js:12))
```javascript
class TerminalManager {
  constructor() {
    this.terminals = new Map();
    this.setupIpcHandlers();  // ← 在构造时自注册
  }
  
  setupIpcHandlers() {
    ipcMain.handle('terminal:create', (event, { terminalId }) => {
      // 需要访问 this.terminals 等内部状态
      this.terminals.set(terminalId, ptyProcess);
    });
  }
}
```

**设计原因：**

| 原因 | 说明 |
|------|------|
| **状态封闭** | TerminalManager 需要维护 [`this.terminals`](frontend/electron/managers/terminalManager.js:7) Map，处理器直接操作内部状态更方便 |
| **双向通信** | 终端需要监听 PTY 输出并主动推送到前端，逻辑复杂 |
| **生命周期** | 终端的事件监听（onData、onExit）需要在 Manager 内部管理 |
| **解耦** | 如果放在 ipcHandlers.js，需要大量暴露 Manager 内部方法，破坏封装 |

**注释说明：**
[`ipcHandlers.js:27`](frontend/electron/handlers/ipcHandlers.js:27) 的注释 `"// 终端 IPC 处理器在 TerminalManager 构造函数中已注册"` 正是说明这一点。

### 🔄 应用启动流程

```
main.js
  ↓ 1. registerIpcHandlers() - 注册 IPC 处理器
  ↓ 2. backendManager.start() - 启动后端服务
  ↓ 3. 等待后端健康检查通过
  ↓ 4. windowManager.createWindow() - 创建窗口
```

### 📝 前端如何使用

```typescript
// 前端代码通过 window.electronAPI 调用
window.electronAPI.restartBackend();
window.electronAPI.windowMinimize();

// 终端功能
window.electronAPI.terminal.create('term-1');
window.electronAPI.terminal.write('term-1', 'ls\n');
window.electronAPI.terminal.onData('term-1', (data) => console.log(data));
```

所有暴露的 API 定义在 [`frontend/src/types/electron.d.ts`](frontend/src/types/electron.d.ts) 中。

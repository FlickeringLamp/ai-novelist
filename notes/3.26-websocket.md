# constructor 是什么
constructor 是 类的构造函数，当你用 new 创建实例时，会自动执行这个函数。


```ts
  constructor(options: WSClientOptions) {
    this.options = {
      clientId: this.generateClientId(),
      reconnectInterval: 3000,
      maxReconnectAttempts: null,
      heartbeatInterval: 30000,
      connectionTimeout: 10000,
      ...options,
    };
  }
```

这是 "默认配置 + 用户自定义" 的合并模式：

constructor(options: WSClientOptions) {
  this.options = {
    // 1. 先放默认值
    clientId: this.generateClientId(),      // 自动生成ID
    reconnectInterval: 3000,                 // 3秒重连
    maxReconnectAttempts: null,              // 无限重试
    heartbeatInterval: 30000,                // 30秒心跳
    connectionTimeout: 10000,                // 10秒超时
    
    // 2. 用用户传入的覆盖默认值
    ...options,
  };
}

...options 是什么？
这是 JavaScript 的 展开运算符（Spread Operator），作用：把 options 对象的属性展开，覆盖前面的默认值。

# this又是什么？

和python的self一样吗？

是的，`this` 和 Python 的 `self` **概念类似**，但工作机制有重要区别。

共同点

都代表**当前实例的引用**，用于访问实例的属性和方法：

**Python：**
```python
class WSClient:
    def connect(self):           # 必须显式声明 self 参数
        self.ws = ...            # 通过 self 访问实例属性
        self.start_heartbeat()    # 通过 self 调用实例方法
```

**TypeScript：**
```typescript
class WSClient {
  connect(): void {              // 不需要声明 this 参数
    this.ws = ...;               // 同样用 this 访问实例
    this.startHeartbeat();       // 调用实例方法
  }
}
```

关键区别

| 特性 | Python `self` | TypeScript `this` |
|------|---------------|-------------------|
| **声明方式** | 必须作为第一个参数显式声明 | 隐式存在，无需声明参数 |
| **命名** | 可以改名叫 `this`、`me` 都行，但约定叫 `self` | 固定关键字，不能改名 |
| **传递方式** | 调用时自动传入：`obj.connect()` → `connect(obj)` | 运行时动态绑定 |

代码中的实际例子

看 [`wsClient.ts`](frontend/src/utils/wsClient.ts:58) 第58-90行：

```typescript
connect(): Promise<void> {
  return new Promise((resolve, reject) => {
    // this 指向当前 WSClient 实例
    if (this.ws?.readyState === WebSocket.OPEN) {
      resolve();
      return;
    }
    
    this.isConnecting = true;              // 访问实例属性
    
    this.ws = new WebSocket(this.getFullUrl());  // 调用实例方法
    
    this.ws.onopen = () => {               // 箭头函数保持 this
      this.reconnectAttempts = 0;          // 这里的 this 仍指向实例
      this.startHeartbeat();
    };
  });
}
```

TypeScript 的 `this` 陷阱

**普通函数会丢失 this**：
```typescript
class Example {
  name = "test";
  
  normalFunction() {
    console.log(this.name);  // OK，this 指向实例
  }
  
  problematic() {
    // 创建一个普通函数作为回调
    setTimeout(function() {
      console.log(this.name);  // ERROR！this 变成 undefined 或 window ———— 这是 JavaScript/TypeScript 的 this 绑定规则导致的。关键是：谁调用了函数，this 就指向谁。
    }, 100);
  }
  
  fixed() {
    // 箭头函数在定义时「捕获」了当前 this
    setTimeout(() => {
      console.log(this.name);  // OK！箭头函数保持 this
    }, 100);
  }
}
```
箭头函数原理：

箭头函数没有自己的 this
它会「借用」定义时外层作用域的 this
相当于闭包，把 this 保存下来了

在代码中，第83行、第92行、第96行、第107行都使用了**箭头函数**，就是为了保持 `this` 指向 `WSClient` 实例。

总结

- **Python**：`self` 是显式参数，调用时自动传递
- **TypeScript**：`this` 是隐式关键字，运行时才确定指向
- **两者作用相同**：都是"我自己"，用来访问自己的东西

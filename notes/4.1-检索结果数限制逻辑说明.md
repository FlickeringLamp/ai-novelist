# 检索结果数限制逻辑说明

## 修改概述

实现了多文件搜索结果数限制（100个），超出时告知AI被折叠了多少个匹配项，并建议使用更精确的检索词。

---

## 1. 单文件搜索（`search_text.py`）

### 修改点：移除20个限制

**修改前：**
```python
for match in matches[:20]:  # 只遍历前20个
    ...
if len(matches) > 20:
    result_text += f"\n... 还有 {len(matches) - 20} 个匹配项"
```

**修改后：**
```python
for match in matches:  # 遍历所有匹配项
    ...
# 不再限制，全部返回
```

**逻辑**：单文件搜索时，显示文件中所有匹配项，不做数量限制。

---

## 2. 多文件搜索（`file_service.py`）

### 2.1 `search_files` 函数（第350行）

**修改**：新增 `max_results` 参数

```python
async def search_files(
    query: str, 
    ignore_file: Optional[str] = None, 
    max_results: int = 100  # 新增默认限制100个
) -> str:
```

调用 ripgrep 时传入限制，ripgrep 内部会做截断。

---

### 2.2 `search_files_for_ai` 函数（第422行）

这是核心逻辑，分三步：

#### 第一步：搜索 max_results + 1 个

```python
max_results = 100
rg_output = await search_files(query, ignore_file, max_results=max_results + 1)  # 搜101个
```

**为什么要搜101个？**

| 搜索结果数 | 含义 | 处理方式 |
|-----------|------|---------|
| ≤ 100 个 | 全部结果都在 | 正常显示，无提示 |
| 101 个 | 实际有更多 | 截断到100个，添加折叠提示 |

> 如果只搜100个，无法判断是"正好100个"还是"超过100个"。搜101个才能确定是否还有更多。

---

#### 第二步：判断是否被截断

```python
match_lines = [line for line in lines if ...]  # 提取有效匹配行
is_truncated = len(match_lines) > max_results  # 超过100个则被截断
```

---

#### 第三步：处理结果并添加提示

```python
# 只保留前100个
for line in lines:
    if count >= max_results:
        continue  # 跳过第101个及之后的
    count += 1
    filtered_lines.append(...)

# 如果被截断，添加提示信息
if is_truncated:
    truncated_count = len(match_lines) - max_results
    result += f"\n\n[提示：检索结果被折叠了 {truncated_count} 个匹配项。建议使用更精确的检索词，或指定具体文件路径以减少检索范围]"
```

## 3. 流程图

```
用户调用 search_text 工具
        │
        ├─► 指定了具体文件？ ───是───► 单文件搜索
        │                                    │
        │                                    ▼
        │                              返回所有匹配
        │                              （无数量限制）
        │
        否（path为null或文件夹）
        │
        ▼
    多文件搜索
        │
        ▼
调用 ripgrep 搜索101个结果
        │
        ├─► 返回 ≤ 100 个？ ───是───► 正常显示全部
        │
        否（返回101个）
        │
        ▼
    结果被截断
        │
        ▼
    显示前100个 + 提示信息
        │
        ▼
[提示：检索结果被折叠了 X 个匹配项。
 建议使用更精确的检索词，或指定具体文件路径以减少检索范围]
```

## 4. 实际效果示例

假设用户搜索常见词"的"，项目中有 **500个匹配**：

### 多文件搜索返回：

```
【工具结果】：在 '根目录' 中找到匹配项：

文件1.txt:10:这是一个的内容
文件1.txt:25:那也是一个的内容
...（共100个匹配）

[提示：检索结果被折叠了 400 个匹配项。建议使用更精确的检索词，或指定具体文件路径以减少检索范围]
```

这样AI就知道结果被截断了，可以：
1. 基于已有的100个结果进行回答
2. 或建议用户使用更精确的检索词重新搜索

# 粗略看了一下，感觉就是给function calling包装了个标准接口？

mcp，感觉就是给function calling包装了一个标准接口
然后所有遵守该接口的客户端，能够兼容所有遵守此接口的服务端
不像传统function calling，只能靠代码移植
mcp可以即插即用
本质上，对于ai模型本身来说，并没有区别，都是工具调用

这些参数是什么意思？

-y

我们的项目能够处理这些参数吗？

-y - 这是 npm 的参数，表示自动确认所有提示（yes）
@modelcontextprotocol/server-filesystem - 这是 npm 包名，一个 MCP 文件系统服务器3. E:\ai-novelist\backend\data - 这是传递给该服务器的参数，指定要共享的文件系统路径

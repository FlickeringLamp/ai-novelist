# 便携版构建问题总结

## 问题列表与解决方案

### 1. 脚本路径问题
**现象**：
```
The term '.<span class="highlight">scripts</span>\build-portable.ps1' is not recognized...
```

**原因**：当前工作目录是 `backend/`，但 `scripts/` 目录在项目根目录

**解决**：从项目根目录运行，并指定 `-ProjectRoot "."`
```powershell
cd F:\盘备份\ai-novelist
.\scripts\build-portable.ps1 -ProjectRoot "."
```

---

### 2. requirements.txt 路径问题
**现象**：
```
找不到 requirements.txt: .'\backend\requirements.txt
```

**原因**：路径参数 `.` 未正确解析，或者路径中包含特殊字符

**解决**：在脚本中使用 `Resolve-Path` 规范化路径
```powershell
$ProjectRoot = (Resolve-Path $ProjectRoot).Path
```

---

### 3. Visual Studio 编译环境检测失败
**现象**：
```
警告: 未找到 Visual Studio，某些包可能需要手动编译
```

**原因**：用户的 VS 2022 安装在 `18\Community`（预览版路径），脚本只检测了 `2022\Community`

**解决**：添加 VS 预览版路径到检测列表
```powershell
$vsPaths = @(
    "${env:ProgramFiles}\Microsoft Visual Studio\18\Community\VC\Auxiliary\Build\vcvars64.bat",
    "${env:ProgramFiles}\Microsoft Visual Studio\2022\Community\VC\Auxiliary\Build\vcvars64.bat",
    # ...
)
```

---

### 4. llama-cpp-python 编译失败
**现象**：
```
Cannot import 'scikit_build_core.build'
```

**原因**：
- `llama-cpp-python` 需要从源码编译
- pip 的构建隔离环境无法找到 `scikit-build-core`

**解决**：
1. 预装构建工具到目标目录
2. 使用 `--no-build-isolation` 参数
```powershell
# 4.6 预装构建工具
& "$BuildDir\python\python.exe" -m pip install `
    --target="$BuildDir\python\Lib\site-packages" `
    scikit-build-core cmake setuptools wheel --quiet

# 5. 安装依赖时禁用构建隔离
& "$BuildDir\python\python.exe" -m pip install `
    --target="$BuildDir\python\Lib\site-packages" `
    -r $RequirementsFile `
    --no-build-isolation `
    --quiet
```

---

### 5. backend 模块导入失败
**现象**：
```
ModuleNotFoundError: No module named 'backend'
```

**原因**：便携环境中 Python 无法找到项目根目录下的 `backend` 模块

**解决**：在 `main.py` 中主动添加项目根目录到 `sys.path`
```python
# 添加项目根目录到 sys.path（确保便携环境中能正确导入 backend）
script_dir = Path(__file__).parent.resolve()
if str(script_dir) not in sys.path:
    sys.path.insert(0, str(script_dir))
```

---

### 6. f-string 反斜杠语法错误
**现象**：
```
SyntaxError: f-string expression part cannot include a backslash
File "...\system_prompt_builder.py", line 160
    return f"[额外文件内容]:\n\n{'\n\n'.join(file_contents)}"
```

**原因**：Python 3.11 的嵌入式版本不支持 f-string 中包含反斜杠转义

**解决**：
1. **方案一**：升级嵌入式 Python 版本到 3.13.9（支持新语法）
2. **方案二**：修改代码避免在 f-string 中使用反斜杠

推荐方案一，修改脚本默认参数：
```powershell
param(
    [string]$PythonVersion = "3.13.9",  # 原为 "3.11.9"
    ...
)
```

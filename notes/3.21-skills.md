输入：


skill: "docx"
args: "(optional)"  // 传给skill的额外参数
输出（对 Claude）：
SKILL.md 的完整内容被注入到当前对话 context 里，Claude 读取后按其指令执行。

本质上，Skill 工具是一个延迟的 context 注入器：


用户: "帮我创建一个 Word 文档"
    ↓
Claude 识别需要 docx skill
    ↓
Claude 调用 Skill("docx")
    ↓
harness: 读取 SKILL.md → 注入到对话
    ↓
Claude: 现在上下文里有了完整的 docx 操作指南
    ↓
Claude 按照 SKILL.md 里的指令去执行任务
类比：Skill 工具 = 动态加载的专业手册

它不产生任何面向用户的输出，输出完全是给 Claude 自己看的——让 Claude 获得原本不在 context 里的专业知识，然后 Claude 用这些知识去完成用户的实际任务。

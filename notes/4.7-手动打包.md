## Electron 绿色免安装包手动制作指南

### 目录结构原理

绿色包 = Electron运行时 + 应用代码 + Python后端

```
绿色包/
├── electron.exe          ← 从 node_modules/electron/dist 复制
├── *.dll                 ← Electron 依赖的 DLL
├── locales/              ← 语言文件
├── resources/
│   └── app/              ← 你的应用代码
│       ├── package.json
│       ├── electron/
│       │   ├── main.js
│       │   ├── preload.mjs
│       │   └── managers/
│       └── dist/         ← vite 构建产物
│           ├── index.html
│           └── assets/
├── python/               ← 便携 Python
└── backend/              ← Python 后端代码
```

---

### 手动操作步骤

#### 1. 准备目录

```powershell
mkdir "dist/绿色免安装版"
cd "dist/绿色免安装版"
```

#### 2. 复制 Electron 运行时

从 `frontend/node_modules/electron/dist/` 复制以下文件到 `dist/绿色免安装版/`：

```
# 必需文件
electron.exe
d3dcompiler_47.dll
dxcompiler.dll
dxil.dll
ffmpeg.dll
icudtl.dat
libEGL.dll
libGLESv2.dll
resources.pak
snapshot_blob.bin
v8_context_snapshot.bin
vk_swiftshader.dll
vk_swiftshader_icd.json
chrome_100_percent.pak
chrome_200_percent.pak

# 必需目录
locales/
```

直接复制dist也行

#### 3. 创建应用目录

```powershell
mkdir resources/app
```

#### 4. 复制前端构建产物

```powershell
# vite 构建产物
copy dist/frontend/* resources/app/ -Recurse
```

#### 5. 复制 Electron 主进程代码

```powershell
copy frontend/electron resources/app/ -Recurse
copy frontend/package.json resources/app/
copy frontend/assets resources/app/ -Recurse
```

#### 6. 修改 package.json

编辑 `resources/app/package.json`，确保有以下字段：

```json
{
  "name": "qingzhu",
  "version": "0.2.0",
  "main": "electron/main.js",
  "type": "module"
}
```

#**删除** `devDependencies` 和 `scripts` 中的开发命令，只保留必要的。

#### 7. 复制 Python 后端

```powershell
copy dist/portable/python .
copy dist/portable/backend .
copy dist/portable/main.py .
copy dist/portable/.env.example .
```
(到根目录)

#### 9. 创建启动脚本（可选）

创建 `启动.bat`：

```bat
@echo off
start "" "%~dp0electron.exe" --app=%~dp0resources\app
```

### 测试运行

双击 `electron.exe` 或 `启动.bat`，应用应该能正常启动。

如果失败，检查：
1. 路径是否正确（特别是 `main.js` 能否找到 `dist/index.html`）
2. Python 路径是否正确识别
3. 查看控制台报错（从 cmd 运行 `electron.exe` 看输出）

![[Pasted image 20260406011322.png]]


**原因**：`node-pty` 是原生 Node 模块（C++ 编译的 `.node` 文件），需要放在正确位置才能被找到。

**解决方案**：

复制 `node_modules` 到绿色包：

```powershell
# 在绿色包目录执行
xcopy "frontend\node_modules" "resources\app\node_modules\" /E /I /H
```

**更好的做法** - 只复制必需的包（节省空间）：

```powershell
# 创建 node_modules 目录
mkdir "resources\app\node_modules"

# 复制 node-pty 及其依赖
xcopy "frontend\node_modules\node-pty" "resources\app\node_modules\node-pty\" /E /I /H
xcopy "frontend\node_modules\nan" "resources\app\node_modules\nan\" /E /I /H
```

xcopy "..\..\frontend\node_modules\node-pty" "resources\app\node_modules\node-pty\" /E /I /H

**验证路径**：
```
resources/app/
├── electron/
├── dist/
├── node_modules/
│   └── node-pty/
│       ├── build/Release/pty.node  ← 关键文件
│       └── package.json
└── package.json
```

如果还不行，检查 `node-pty` 是否针对当前 Electron 版本编译过。在 frontend 目录执行：

```powershell
npx electron-rebuild -f -w node-pty
```

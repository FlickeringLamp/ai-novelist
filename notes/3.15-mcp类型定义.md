困惑：
```ts
export interface MCPData {
  selectedId: string | null;
  config: {
    [serverId: string]: MCPServerConfig;
  };
}
```
[serverId: string]: MCPServerConfig 叫做索引签名，意思是：
config 是一个对象
可以有任意多个键值对
键是 string 类型（如 "server-1", "server-2"）
值是 MCPServerConfig 类型

那么，为什么MCPData就不能是
MCPData {
  selectedId1: string | null;
  config-1: {
    [serverId: string]: MCPServerConfig;
  };
  selectedId2: string | null;
  config-2: {
    [serverId: string]: MCPServerConfig;
  };
}
既然selectedId和config都描述了类型
又没有限定条件
这样出现的多个元素又该如何？
config能认为定义的是个模板，实际元素有多个
整个MCPData为什么不能认为定义了个模板，内部有很多selectedId？

类型的层级：

核心区别
export interface MCPData {
  // 第1层：MCPData 本身是一个对象类型
  selectedId: string | null;           // ← 属性值 = 单个字符串/null
  config: {                            // ← 属性值 = 另一个对象（内部可有多个键）
    [serverId: string]: MCPServerConfig;  // ← 这是 config 对象的"内部结构"
  };
}
为什么不能直接扩展 selectedId？
TypeScript 的属性名是固定的，不能动态生成：

// ❌ 这种写法在 TS 中是非法的
interface MCPData {
  [动态生成: string]: string | null;  // 不能这样定义 selectedId1, selectedId2...
  config: {...};
}
属性名（selectedId、config）在类型定义时就固定死了，不像 config 内部的 [serverId: string] 那样可以动态变化。


索引的语法和写法又是什么呢？

TypeScript 索引签名（Index Signature）完整语法
基本语法

  [键名: 键类型]: 值类型

1. 差异对比视图的逻辑：

差异对比视图和普通标签页确实是同一个标签，只是用 diffModeTabs 这个独立标记来控制显示方式。

数据结构
editorState 中有这些关键字段：

{
  tabBars: { bar1: { tabs: [], activeTabId: null } },  // 标签栏（只存ID）
  currentData: {},     // 当前编辑内容（右侧/修改后）
  backUp: {},          // 备份内容（左侧/原始内容）
  diffModeTabs: {},    // ⚠️ 差异模式标记 { [filePath]: true }
  checkpointPreviewTabs: {},  // 存档预览标记
}

显示逻辑
UI 根据 diffModeTabs[filePath] 决定渲染方式：

true → 渲染差异对比视图（左右对比）
undefined/false → 渲染普通编辑器

手动关闭时，没有及时清理这个标识，导致了后续打开时，依然显示错误的视图

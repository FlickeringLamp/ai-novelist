# 问题描述：
我们有个加载/卸载文件功能
但是我们觉得这个短板太明显
系统提示词过多，干扰AI的思路
而且不能超过5个文件，超过后提示词过度膨胀，不得不清理订阅

这是我的新想法：
系统提示词之所以能够承接这么多的可变信息，作为AI获取环境可变信息的重要源头
是因为graph里，它被独立于整个状态之外单独维护
每次调用时拼接起来
（状态之内的信息基本不可变，因为它们会被记录到存档点固定）

我们的解决方案就很明确了：
再维护一个可变消息
拼接到末尾
形成“系统提示词” + “核心状态消息” + “末尾附加消息”

消息类型是human message
这样我们就可以订阅几十个文件，而不污染AI的系统提示词
大部分原系统提示词构建器控制的部分，都可以放在这里面（比如rag结果，skills列表，文件列表等）

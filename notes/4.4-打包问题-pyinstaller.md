# PyInstaller 打包问题：ModuleNotFoundError: No module named 'git'

日期: 2026-04-04

## 问题描述

执行 `pyinstaller main.spec` 打包后，运行 `dist/backend/ai-novelist.exe` 报错：

```
ModuleNotFoundError: No module named 'git'
```

错误发生在导入 `backend/settings/initializer.py` 中的 `from git import Repo` 时。

## 根本原因

**PyInstaller 没有安装在虚拟环境中**，使用了全局的 PyInstaller，导致它无法找到虚拟环境中安装的 GitPython 包。

运行环境检查：
```powershell
# Python 解释器路径 (虚拟环境)
F:\盘备份\ai-novelist\.venv\Scripts\python.exe

# PyInstaller 位置 (不存在于虚拟环境)
WARNING: Package(s) not found: pyinstaller
```

## 解决方案

### 1. 在虚拟环境中安装 PyInstaller

```powershell
pip install pyinstaller
```

### 2. 确保 main.py 设置 git 可执行文件路径

程序需要在导入 GitPython 之前，设置 `GIT_PYTHON_GIT_EXECUTABLE` 环境变量：

```python
import os
import sys
from pathlib import Path

# 配置 GitPython 使用打包的 git 可执行文件
if getattr(sys, 'frozen', False):
    base_path = Path(sys._MEIPASS)
    git_exe = base_path / 'bin' / 'PortableGit' / 'mingw64' / 'bin' / 'git.exe'
    if git_exe.exists():
        os.environ['GIT_PYTHON_GIT_EXECUTABLE'] = str(git_exe)
        os.environ['GIT_PYTHON_REFRESH'] = 'quiet'
```

### 3. 确保 PortableGit 已解压

```powershell
cd bin
7z x PortableGit-2.53.0-64-bit.7z.exe -ogit -y
```

git 可执行文件位置：`bin/PortableGit/mingw64/bin/git.exe`

### 4. 打包配置

在 `main.spec` 中，只需将 `'git'` 添加到 `packages` 列表：

```python
packages = [
    # ... 其他包
    'git',  # GitPython 会自动收集
]
```

无需额外的手动子模块收集，PyInstaller 会自动处理。

## 注意事项

1. **必须在虚拟环境中运行 PyInstaller**，否则找不到虚拟环境安装的包
2. 打包后的 git 位于 `_internal/bin/PortableGit/mingw64/bin/git.exe`
3. 需要在 main.py 中设置环境变量才能让 GitPython 找到打包的 git

## 更新到 requirements.txt

```
pyinstaller==6.19.0
```

## 完整打包流程

```powershell
# 1. 确保在虚拟环境中
cd f:/盘备份/ai-novelist

# 2. 确保 PortableGit 已解压 (在 bin/PortableGit)

# 3. 安装 PyInstaller (如果尚未安装)
pip install pyinstaller

# 4. 执行打包
pyinstaller main.spec --clean

# 5. 测试运行
./dist/backend/ai-novelist.exe
```

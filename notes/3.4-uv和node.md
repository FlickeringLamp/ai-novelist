# 问题描述
使用uv和node，无法直接导入各个mcp服务器的readme文档中json，
而且要求用户填写uv和node，也容易让用户困惑（毕竟大部分mcp教程等都是教填写uvx和npx）
但是我们的项目使用uv和node，不采用npx和uvx，因为前者更好控制，无论是下载地址还是镜像源，或者持久化保存

# 解决方案
对外，依然让用户填写npx，uvx，
导入的json也是npx，uvx
包括保存的配置文件也使用npx和uvx，
直到将配置信息塞入langchain的mcp client之前，
或者下载/删除时，
都要完成npx-node,uvx-uv命令的转化

————————
其他：

npm exec命令，会尝试优先下载包，然后再使用
uvx同理，应该用uv tool run

npm exec 或 npx：会检查包版本、下载包（如果不存在）、执行包。需要联网。
直接运行 node_modules/.bin/下的可执行文件：直接执行本地已安装的包，不需要联网。

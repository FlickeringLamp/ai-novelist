两种主题色用法的区别：

1. Tailwind CSS 的 theme-green
<div className="text-theme-green">

工作原理：

在 index.css 中定义了 CSS 变量映射：
--color-theme-green: var(--color-green);

Tailwind 在编译时把 text-theme-green 转成 color: var(--color-theme-green)
浏览器渲染时，CSS 变量解析为实际颜色值
特点：

✅ 只能在 JSX 的 className 中使用
✅ 支持 CSS 伪类（hover、focus 等）
❌ 无法获取具体的十六进制值（如 '#34eb5c'）
2. useTheme() 后访问 .green
const { theme } = useTheme();
const color = theme.green; // '#34eb5c'

工作原理：

从 React Context 获取 JS 对象
直接拿到字符串 '#34eb5c'
特点：

✅ 可以在 JS/TS 代码中使用
✅ 可以拼接字符串（如 theme.green + '40' 加透明度）
❌ 无法在 className 中使用
对比示例
场景	用哪种	代码
普通元素着色	Tailwind	className="text-theme-green"
动态计算颜色	useTheme	theme.green + '40'
Monaco Editor 主题	useTheme	foreground: theme.green
xterm.js 主题	useTheme	foreground: theme.green
hover 效果	Tailwind	className="hover:text-theme-green"
两者关系
它们底层是同一套颜色：

// ThemeContext.tsx - JS 对象
const defaultTheme = {
  green: '#34eb5c',
  // ...
}

// index.css - CSS 变量
:root {
  --color-green: #34eb5c;
  /* ... */
}

// Tailwind 映射
@theme {
  --color-theme-green: var(--color-green); /* → #34eb5c */
}

简单说：

CSS/Tailwind → 用于 UI 布局、className 样式
useTheme() → 用于 JS 库配置（Monaco、xterm 等需要十六进制值的场景）

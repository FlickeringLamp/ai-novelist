# ripgrep 在 Windows 上的 asyncio 子进程问题

## 问题描述

搜索功能在 Windows 上无法正常工作，日志显示：
```
2026-04-04 18:49:04,556 - backend.file.ripgrep_service - ERROR - 搜索失败: 
2026-04-04 18:49:04,556 - backend.api.file_api - INFO - 搜索关键词: 测试, 搜索结果数量: 0
```

## 根本原因

在 Windows 上使用 `asyncio.create_subprocess_exec()` 会抛出 `NotImplementedError`：

```
File "...\Python313\Lib\asyncio\base_events.py", line 539, in _make_subprocess_transport
    raise NotImplementedError
NotImplementedError
```

这是因为 Python 的 asyncio 在 Windows 上默认使用 `SelectorEventLoop`，而它不支持子进程操作。

## 解决方案

使用 `run_in_executor()` 包装同步的 `subprocess.run()` 来替代异步子进程调用：

```python
# 错误的方式（Windows 不支持）
process = await asyncio.create_subprocess_exec(*cmd, ...)
stdout, stderr = await process.communicate()

# 正确的方式
loop = asyncio.get_running_loop()
result = await loop.run_in_executor(None, lambda: subprocess.run(cmd, capture_output=True, text=True))
output = result.stdout
```

## 代码变更

文件: `backend/file/ripgrep_service.py`

将 `asyncio.create_subprocess_exec()` 替换为 `run_in_executor()` + `subprocess.run()` 的组合。

## 参考

- Python asyncio 文档: https://docs.python.org/3/library/asyncio-subprocess.html
- 注意: Windows 上需要使用 `ProactorEventLoop` 才能支持子进程，但 `run_in_executor` 是更简单的跨平台方案

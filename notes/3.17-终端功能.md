### 1. 简化导入路径

**没有这个文件时**，其他组件需要这样导入：
```typescript
import { TerminalPanel } from './components/terminal/TerminalPanel';
import { XTerm } from './components/terminal/XTerm';
```

**有了这个文件后**，可以简化为：
```typescript
import { TerminalPanel, XTerm } from './components/terminal';
// 或者
import { TerminalPanel } from '@/components/terminal';
```

### 2. 封装内部结构

假设将来你把 `XTerm.tsx` 重命名为 `TerminalCore.tsx`，只要 `index.ts` 里改成：
```typescript
export { TerminalCore as XTerm } from './TerminalCore';
```
外部所有导入 `XTerm` 的地方都不需要修改。

### 3. 集中管理公共 API

如果终端组件有内部使用的工具函数，但不想暴露给外部，就可以只在 `index.ts` 中导出需要公开的组件：
```typescript
// 内部使用，不导出

// 只导出公共组件

```

order 是 react-resizable-panels 库中 Panel 组件的一个属性，用于指定面板在面板组中的顺序。

作用说明
<PanelGroup direction="vertical">
  {/* 编辑器在上 */}
  <Panel order={1} ... />   // order=1，排在第一个

为什么要用 order？
在 react-resizable-panels 中：

控制渲染顺序 - order 值小的面板排在前面（垂直布局就是排在上面，水平布局就是排在左边）

条件渲染时保持顺序 - 注意看代码中终端面板是条件渲染的（{isTerminalVisible && (...)}）：

当 isTerminalVisible 为 false 时，只有编辑器 Panel 存在
当 isTerminalVisible 为 true 时，两个 Panel 都存在
防止顺序错乱 - 如果不指定 order，当条件切换时，React 可能会因为组件挂载/卸载顺序导致面板位置错乱

## `useCallback` 的主要作用

`useCallback` 是 React 的一个 Hook，用于**缓存函数引用**，避免每次渲染都创建新的函数实例。

### 核心用途：

1. **防止子组件不必要的重渲染** - 当函数作为 props 传递给子组件时，稳定的函数引用可以避免子组件重复渲染

2. **避免 useEffect 无限循环** - 当函数需要在 `useEffect` 依赖数组中使用时，`useCallback` 能确保函数引用稳定

### 在你代码中的实际例子：

**`createTerminal`** (第21行)：
```typescript
const createTerminal = useCallback(() => {
  const id = `terminal-${Date.now()}-${terminalCounter.current++}`;
  setTerminals(prev => [...prev, { id, name: `终端 ${terminalCounter.current - 1}` }]);
  setActiveTerminalId(id);
}, []); // 空依赖数组，函数永远不会变
```
这个函数被放入第63行的 `useEffect` 依赖数组中。如果不使用 `useCallback`，每次渲染都会创建新函数，导致 `useEffect` 无限循环触发。

**`closeTerminal`** (第30行)：
```typescript
const closeTerminal = useCallback((id: string) => {
  // 使用 activeTerminalId
}, [activeTerminalId]); // 只有 activeTerminalId 变化时才重新创建
```

**`closeAllTerminals`** (第51行)：
```typescript
const closeAllTerminals = useCallback(() => {
  window.electronAPI?.terminal.killAll();
  setTerminals([]);
  setActiveTerminalId(null);
}, []);
```

### 使用原则：

| 场景 | 是否需要 `useCallback` |
|------|----------------------|
| 函数传给子组件 | ✅ 需要 |
| 函数在 `useEffect` 依赖数组中 | ✅ 需要 |
| 函数只在当前组件内部使用 | ❌ 不需要 |
| 配合 `React.memo` 的子组件 | ✅ 需要 |

简单说：`useCallback` 就是为了保持函数引用稳定，避免不必要的重渲染和副作用重复执行。

监听机制
关键点解析
1. useEffect 只执行一次，但事件监听是持久的

useEffect 的 [] 依赖表示：组件挂载时执行一次
但 window.addEventListener('keydown', ...) 会把回调函数注册到 DOM 的 window 对象 上
这个监听会一直存在，直到组件卸载时调用 removeEventListener
2. 事件冒泡机制

用户按键
    ↓
触发 window 的 keydown 事件
    ↓
执行 handleKeyDown 回调

只要页面处于焦点状态，所有键盘事件都会冒泡到 window 对象。

3. 为什么能获取到最新的 handleToggleTerminal？

虽然 useEffect 只执行一次，但 handleToggleTerminal 是闭包捕获的引用：

const handleToggleTerminal = () => {
  setIsTerminalVisible(prev => !prev);  // 使用函数式更新，始终是最新状态
};

setIsTerminalVisible 是 React 的 setter，始终引用同一个函数
所以即使 useEffect 只执行一次，也能正确调用状态更新

React ref 绑定机制
ref={containerRef} 是 React 的特殊语法：

首次渲染时：React 创建 DOM 元素后，自动将元素赋值给 containerRef.current
卸载时：React 自动将 containerRef.current 设为 null



完整时间线
组件首次渲染
    ↓
const containerRef = useRef(null)  // current = null
    ↓
return <div ref={containerRef} />  // JSX 中的 ref 属性
    ↓
React 创建真实 DOM 元素
    ↓
React 自动执行：containerRef.current = domElement
    ↓
浏览器绘制完成
    ↓
useEffect 执行                          // 此时 current 有值了！
    ↓
const container = containerRef.current   // ← 可以安全使用了

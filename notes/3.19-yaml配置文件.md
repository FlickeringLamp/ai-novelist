Python 数据对象（dict/list/str/int 等）

       ↓ 序列化
JSON:  json.dump()  →  .json 文件
YAML:  yaml.dump()  →  .yaml 文件
       ↑ 反序列化
Python 数据对象（完全一样）

数据兼容性
JSON 和 YAML 在数据模型上完全兼容：

Python 类型	JSON	YAML
dict	{"a": 1}	a: 1
list	[1, 2, 3]	- 1\n- 2\n- 3
str/int/float/bool/None	相同	相同
结论
✅ 原来存进 JSON 的 Python 数据 → 现在存进 YAML → 读出来完全一样
✅ 所有业务代码无需改动，只跟 Python 对象打交道
✅ 只有 config.py 里的文件读写逻辑改变
中间存储格式变了，但数据本身和业务代码逻辑完全不变。

新的错误：
这个错误是 LangGraph 框架抛出的 InvalidUpdateError，具体信息是 “Ambiguous update, specify as_node”。它发生在你尝试通过 aupdate_state 更新图的状态时，没有提供 as_node 参数，导致框架无法确定这次更新是来自哪个节点，从而无法正确执行状态转换。


解决方法

在调用 aupdate_state 时，添加 as_node 参数，指明这次更新是模拟哪个节点产生的。例如：
python

await graph.aupdate_state(
    previous_config,
    values={"messages": [new_msg]},
    as_node="your_node_name"   # 替换为图中实际存在的节点名，比如 "call_llm"
)

从某个特定节点
当你打电话时update_state使用指定节点的编写工具(包括 reducersdefroners)来应用值。检查点会将该节点记录为已生成更新,执行从该节点的后续步骤中恢复。 默认情况下,LangGraph 会推断as_node来自检查点的版本历史记录。从某个检查点进行分叉时,这种推断几乎总是正确的。 指定as_node明确时间:

    Parallel branches并行分支:多个节点在同一步骤中更新状态,而 LangGraph 无法确定最后哪个节点(InvalidUpdateError)
    No execution history无执行历史记录:在新线程上设置状态(在测试中常见testing)。
    Skipping nodes跳过节点:设置as_node稍后节点,使图表认为节点已经运行。

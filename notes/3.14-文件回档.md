### 问题现象
调用存档点差异 API (`/api/checkpoints/diff/{commit_hash}`) 时，返回结果中的 `change_type` 字段为 `null`，导致前端无法正确识别文件变更类型。

### 问题原因

通过分析 GitPython 源码（`/home/denghuo/文档/ai-novelist/backend_env/lib/python3.12/site-packages/git/diff.py`）发现：

1. **第 168 行**：原代码使用了 `create_patch=True` 参数
   ```python
   diff = commit.diff(parent_commit, create_patch=True, unified=3)
   ```

2. **GitPython 源码第 648-666 行**：当使用 `create_patch=True` 时，`Diff` 对象的 `change_type` 参数被硬编码为 `None`

3. **第 428 行**：`change_type` 的类型定义为 `Optional[Lit_change_type]`，允许为 `None`

4. **`change_type` 只有在使用非 patch 格式时才会被正确解析**（第 692-696 行的 `_handle_diff_line` 方法）

因此，`create_patch=True` 与 `change_type` 属性是互斥的。

### 解决方案

**移除 `create_patch=True` 参数**：使用默认的 diff 格式，这样 `item.change_type` 就能正确返回 'A'（新增）、'D'（删除）、'M'（修改）等值

**核心原理：**GitPython 的 diff() 方法有两种模式：

Raw 模式（默认）：简洁输出，适合获取文件列表和 blob 对象
Patch 模式（create_patch=True）：完整 diff 输出，适合生成 patch 文件
我们只需要获取文件内容和变更信息，使用 raw 模式即可，不需要 create_patch 或 unified 参数。

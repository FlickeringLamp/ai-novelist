# 代码组织逻辑：

## 一级文件：
1. settings.py : 核心文件1,用于管理env,config
2. initializer.py: 核心文件2,用于初始化时创建所有需要的文件，或者启动时检查设置文件的完整性

## 二级文件：
1. paths.py：用于在开发和生产环境下，找出配置文件，环境变量文件的路径
2. env_manager.py: 提供管理环境变量的各个函数

### 三级文件：
1. env.py： 提供环境变量项和提供商名的映射
4. tools.py: 提供工具id,工具名，工具描述的映射


规则：
backend/settings/ 文件夹以外，任何模块都只能从settings实例里获取属性和方法
不允许直接读取具体的env和tool映射数据，也不允许自行修改配置文件和环境变量文件

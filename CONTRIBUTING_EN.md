# Contributing Guide

We warmly welcome and appreciate your contributions to this project!

---

## How to Contribute

### 1. Reporting Bugs

If you discover any bugs, please submit a bug report in GitHub Issues. Please provide as much detail as possible, including:

*   Steps to reproduce
*   Expected behavior
*   Actual behavior
*   Error messages (if any)
*   Your operating system and environment information

### 2. Submitting Feature Requests

If you have new feature suggestions, please submit a feature request in GitHub Issues as well. Describe your ideas in detail and how they would improve the project.

### 3. Submitting Code

**Important: Please create an Issue for discussion before starting coding**

To avoid wasted effort, we recommend the following workflow:

#### Step 1: Create an Issue
Before submitting code, please create an Issue describing the bug you want to fix or the feature you want to implement. This helps:
- Confirm whether the issue/feature has already been addressed
- Discuss implementation plans to avoid misalignment with the project direction
- Receive feedback and suggestions from maintainers

#### Step 2: Fork the Repository
Fork this project's repository to your own GitHub account.

#### Step 3: Clone the Repository
Clone your forked repository to your local machine:
```bash
git clone git@github.com:FlickeringLamp/ai-novelist.git
cd ai-novelist
```

#### Step 4: Create a Branch
Create a new branch for your feature or bug fix:
```bash
git checkout -b feature/your-feature-name
```

#### Step 5: Make Changes
Write your code and test it.

#### Step 6: Commit Changes
Commit your changes:
```bash
git commit -m "feat: Add your feature"
```

#### Step 7: Push Branch
Push your changes to your forked repository:
```bash
git push origin feature/your-feature-name
```

#### Step 8: Create Pull Request
Create a Pull Request on GitHub from your fork branch to the main branch of this project. Please:
- Reference the related Issue in the PR description (e.g., `Fixes #123`)
- Describe your changes in detail

## Contribution Rules

To ensure the healthy development of the project, please follow these basic rules:

### License Compatibility
All submitted code must be original or explicitly compatible with the [MIT License](LICENSE). **Strictly prohibited from introducing any GPL, AGPL, or other Copyleft licensed code**, as these licenses may impose restrictions on the project.

### Code Quality
Please ensure your code has good readability and maintainability.

### Test Coverage
If possible, please add corresponding tests for your changes.
